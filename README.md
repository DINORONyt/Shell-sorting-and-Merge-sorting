# Лабораторная работа №3: Алгоритмы и анализ сложности
## Вариант 17: Сортировка слиянием и сортировка Шелла

### Описание проекта
Данный проект представляет собой Windows Forms приложение на C#, разработанное в рамках лабораторной работы №3 по дисциплине "Алгоритмизация и программирование". Программа реализует два классических алгоритма сортировки, проводит экспериментальное исследование их производительности, подсчитывает элементарные операции и визуализирует результаты сравнения. Бизнес-логика вынесена в отдельную библиотеку, интерфейс отделён от вычислений, проект покрыт модульными тестами.

### Функциональность

**Задача 1 (Базовая)** — Сортировка слиянием (Merge Sort)  
Реализация алгоритма "разделяй и властвуй" со сбором метрик: время выполнения, количество сравнений и перестановок. Проверка корректности результата.

**Задача 2 (Средняя)** — Сортировка Шелла (Shell Sort)  
Реализация усовершенствованной сортировки вставками с уменьшающимся шагом (gap). Сбор аналогичных метрик производительности.

**Задача 3 (Расширенная)** — Сравнительный анализ и визуализация  
Автоматическое тестирование обоих алгоритмов на массивах разного размера (100, 500, 1000, 5000, 10000). Вывод результатов в таблицу и графическое отображение исходного массива.

### Установка и запуск

#### Требования
- .NET 8.0 SDK
- Visual Studio 2022 (с компонентом ".NET desktop development")

#### Запуск через Visual Studio
1. Откройте файл решения `Lab03Variant17.sln`
2. Выберите проект `WinFormsApp3` как стартовый
3. Нажмите `F5` или кнопку "Start"

#### Запуск через командную строку
```bash
dotnet restore
dotnet run --project WinFormsApp3
```

### Использование приложения

При запуске открывается окно с четырьмя вкладками:

**Вкладка 1: Генерация массива**
1. Введите размер массива N (допустимо: 100–10000)
2. Нажмите "Сгенерировать массив"
3. Приложение создаст массив случайных чисел в диапазоне [-1000, 1000]

**Вкладка 2: Запуск алгоритмов**
1. Нажмите "Запустить" в группе "Сортировка слиянием"
2. Нажмите "Запустить" в группе "Сортировка Шелла"
3. Для каждого алгоритма отобразятся: время выполнения (мс), сравнения, перестановки и статус отсортированности

**Вкладка 3: Сравнение результатов**
1. Нажмите "Сравнить алгоритмы на разных размерах"
2. Приложение автоматически протестирует оба алгоритма на массивах разного размера
3. Результаты отобразятся в таблице с колонками: Размер, Алгоритм, Время, Сравнения, Перестановки

**Вкладка 4: Визуализация**
1. Нажмите "Показать визуализацию"
2. В `PictureBox` отобразится столбчатая диаграмма исходного массива (масштабирование по высоте)

### Обработка ошибок

Приложение включает многоуровневую валидацию согласно ТЗ:
- Проверка формата ввода (только целые числа)
- Ограничение диапазона размера массива: от 100 до 10000 элементов
- Запрет запуска алгоритмов на пустом массиве
- Требование предварительного запуска обоих алгоритмов перед сравнением
- Вывод сообщений через `MessageBox` с иконкой предупреждения
- Архитектурная защита: методы валидации вынесены в библиотеку, что предотвращает дублирование кода и связывание UI с логикой
